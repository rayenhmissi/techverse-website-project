introduction

previously,OS is developed in a single block of code.MS DOS is one of them.but this was a problem for developers in terms of testing and debuging and this make the OS less modular but in modern OS,the structure become easier to understand and more efficiant. let's take a look of the structure of linux as an example for this lesson.

table of content

  1. kernel
  2. system libraries
  3. shells
  4. hardware layer
  5. user applicatiosns
  6. utilities
  1. kernel

    2. the kernel is the core of linux,it is responsible for process management ,memory management,communication between hardware and software and file management.linux is a monolithic OS that allocate all the resources needed in a single bloc of code .

  2. system libraries

    3. system libraries are prebuilt functions to allow developers make apps easier and optimise them and allow them to interact with the kernel to run them efficiantly.GNU C library is one of these libraries.

  3. shells

    4. shells are essential to use linux. it allows you writing command to exvute some tasks like file management.bash is one of these shells to write commands in linux.

  4. hardware layer

    5. hardware layer contain all the hardware components to let the computer run well and the kernel can access them and manage their performance.

  5. user applications

    6. this contains all the applications that users need for daily tasks like a web browser or a notepad.

  6. utilities

    7. these utilities are responsible for system settings like network settings,file management....
